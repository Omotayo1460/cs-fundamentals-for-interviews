## 🚀 How to use this

1. **`Use this template`** / Fork — so you can add your own notes and track what you've revised.
2. Skim a subject the night before an interview; each answer is intentionally short.
3. Cover the answer, ask yourself the question out loud, then check. Repeat the ones you miss.
4. Add company-specific questions you're asked under the right subject — contributions welcome!

## 🎯 Most-asked, all subjects (quick-fire)

A taste — full answers inside each file:

- **OS:** Difference between process and thread? What causes a deadlock and how to prevent it?
- **DBMS:** What is normalization? Explain ACID. Clustered vs non-clustered index?
- **CN:** TCP vs UDP? What happens when you type a URL and press Enter?
- **OOP:** Four pillars? Overloading vs overriding? Abstract class vs interface?
