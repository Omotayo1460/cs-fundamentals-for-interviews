# 🤝 Contributing

Help fellow students ace their interviews! This is a beginner-friendly repo and a great
place for your **first open-source pull request**. 🎉

## Ways to contribute

- ➕ Add an interview question + concise answer to any subject file.
- 🏢 Add a **company-specific** question you were actually asked (mention the company).
- 📘 Add a new subject (e.g. `system-design.md`, `dsa-quick-revision.md`) and link it from
  the [README](README.md) table.
- ✏️ Sharpen or correct an existing answer; fix typos.

## Style guide

- Keep answers **short and interview-ready** — 1 to 4 sentences. Depth over length is the
  enemy here; people revise these the night before.
- Use the existing format: `**N. Question?**` followed by the answer.
- Prefer plain language and concrete examples.
- Keep each subject's back-link line at the top intact.

## Before you open a PR

Run the link checker so CI stays green:

```bash
python tools/check_links.py
```

Then commit, push to your fork, and open a PR describing what you added. Thank you! ❤️
